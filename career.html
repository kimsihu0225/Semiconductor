Process Engineer

하는 일: 공정 조건 최적화, defect 개선, 수율 향상.

필요 지식: 8대 공정, defect, 통계, data interpretation.

중요 키워드: CD, uniformity, SPC, yield.

면접 포인트: 공정 간 연계와 문제 해결 관점 설명.

Equipment Engineer

하는 일: 장비 안정화, PM, trouble shooting, uptime 관리.

필요 지식: vacuum, plasma, RF, sensor, mechanical system.

중요 키워드: PM, chamber, calibration, MTBF.

면접 포인트: 장비 구조와 이상 원인 추적 방식.

Yield Engineer

하는 일: 수율 분석, 불량 원인 분류, data 기반 개선.

필요 지식: defect analysis, 통계, 공정 흐름 이해.

중요 키워드: fail bit, defect density, pareto.

면접 포인트: 데이터에서 원인을 좁혀 가는 사고 방식.

Test Engineer

하는 일: 테스트 조건 설계, 양불 판정, 패키지 이후 검증.

필요 지식: 회로 기초, tester, 신호, 품질 개념.

중요 키워드: probe, binning, coverage, reliability.

면접 포인트: 테스트 목적과 생산 품질의 연결.

Automation / Software Engineer

하는 일: 장비 자동화, 데이터 수집, 공정 시스템 연계.

필요 지식: Python, C/C++, PLC/SECS-GEM, 데이터 처리.

중요 키워드: MES, automation, log, integration.

면접 포인트: 소프트웨어가 생산성과 품질에 미치는 영향.
